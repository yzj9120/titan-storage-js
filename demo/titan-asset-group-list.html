listDirectoryContents

This method allows you to fetch a list of asset groups with pagination. It ensures that all provided inputs, including the page number, parent ID, and page size, are valid before making the server request.

Function Signature

const result = await listDirectoryContents(page, parent, pageSize);

This method communicates with the server to retrieve asset groups based on the specified page, parent group ID, and the number of groups to display per page.

Example

const result = await listDirectoryContents(1, 0, 10);
if (result.code === 0) {
    console.log(result.data); // Success, list of asset groups
} else {
    console.error(result.message); // Failure

Parameters

  • page: Number - The page number. This is a required parameter and must be a positive integer.
  • parent: Number - The ID of the parent group. This is an optional parameter, and defaults to the root group if not provided.
  • pageSize: Number - The number of items to display per page. This must be a valid positive integer, typically between 1 and 100.

Returns

This function returns a Promise which resolves to an object containing:

  • code: A number representing the result of the operation (0 for success, other values indicate failure).
  • data: A list of asset groups if the operation is successful.
  • msg: A string describing the outcome of the operation, especially in case of errors.

Error Handling

If the request to retrieve the asset group list fails, the code will be non-zero, and the message will provide details about the error, such as an invalid page number, parent ID, or server-related issues.

Important Note

Ensure that the parameters page, parent, and pageSize are valid before calling this method to avoid unnecessary errors. The method uses validators to check the input values, and it will return any validation errors if the inputs are invalid.
