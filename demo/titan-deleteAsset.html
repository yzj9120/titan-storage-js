deleteAsset

This method allows you to delete a specific asset by its ID. It validates the input and communicates with the server to perform the delete operation.

Function Signature

const result = await deleteAsset(options);

Example


const result = await deleteAsset({ assetId: bafybeibe4awofqwksm46xr64yfhz7cca2vo5fj723qga6m5rhgum6uu3le, areaId: [“SouthKorea”, “Japan”] });
if (result.code === 0) {
    //success
} else {
    //fail
}

Parameters

  • assetId: String - TThe CID of the asset to be deleted. This is a required parameter. Cid located under AssetOverview object.
  • areaId: Array<String> - An array of optional node region IDs associated with the asset, which by default deletes files under all node regions. Area_ids located under the AssetOverview object. Multiple choices are available, indicating the need to delete files under the node and not delete files in other node areas.

Returns

This function returns a Promise which resolves to an object containing:

  • code: A number representing the result of the operation (0 for success, other values indicate failure).
  • data: The result of the delete operation if successful.
  • msg: A string describing the outcome of the operation, especially in case of errors.

Error Handling

If the deletion fails, the code will be non-0, and the message will provide details about the error, such as invalid asset ID or server-related issues.

Important Note

Ensure that the assetId is valid before calling the method to avoid unnecessary errors. The method uses validators to check the input values, and it will return any validation errors if the inputs are invalid.
